CS695 - Assignment 2 Rubrics

  1. A) DIY Hypervisor - 4 Marks
    You need to see the part1a.pdf file to grade this
  1. Flowchart - 2 Marks
  1. At least ten correct logical actions - 2/2 Marks
    The actions should be descriptive and not just API calls.
  2. Written API calls only  - 0.5/2 Mark
  1. KVM API explanation - 2 Marks
    e.g.,
    1) Set up the physical memory for the guest OS… To assign the memory to the guest OS, the KVM_YYY_ZZZ ioctl call is used.
    KVM_YYY_ZZZ:
            Brief description of inputs to the ioctl call
            Functionality that the ioctl call provides, etc.

  1. B) Hypercalls: How can I help you? - 5 Marks
  1. HC_print32bit works correctly - 1 Mark
  1. The first exit count is not 14 - 0.5/1 Mark
    This means the print32bit is using multiple print8bit internally
  1. HC_numExits works correctly - 1 Mark
  2. HC_printStr works correctly - 1 Mark
  1. Values don’t differ by 2 - 0/1 Mark
  1. HC_numExitsByType correctly - 1 Mark
  1. If the address is hard coded and/or the address GVA sent to the userspace program is not translated to HVA (if this is the implementation) - 0.5/1 Mark
    Line number 219-224 in simple-kvm.c shows the translation required.
  2. The IO in count doesn’t represent KVM_EXIT_IO_IN - 0/1 Mark
    To verify this, look into the code and make sure the in count is only updated in KVM_EXIT_IO_IN. [The values in the sample output of IO in and out can change based on implementation but should represent the actual IO in-out, i.e., what assembly instruction is being used. (out or in)]
  1. HC_gvaToHva works correctly - 1 Mark
  1. The GVA where the payload is stored is not translated to HVA
    Line number 234-239 in simple-kvm.c shows the translation required.  - 0.5/1 Mark
  2. The invalid addresses are not being detected in 32-bit paging and 64-bit paging - 0/1 Mark

  1. Build the matrix cloud - 6 Marks
  1. If all test cases are passing - 6/6 Marks
    Otherwise, deduct 1 mark for each test case.
  1. Test case 1: FRAC_A = 7 & FRAC_B = 3 is working correctly
  2. Test case 2: FRAC_A = 9 & FRAC_B = 1 is working correctly
  3. Test case 3: FRAC_A = 0 & FRAC_B = 10 is working correctly

        

NOTE: There should be no sleep() calls in the code. If present, consider the test cases as failed and 2.1 as void. Moreover, there should be only one CPU at use [can be checked using htop/top]

  1. If all the test cases do not pass, check if matrix-b works. That is, check if the timer can kick the vcpu out of running state at every QUANTUM. Ensure that the rax value changes, indicating the VM is running alternatively. -  3/6 Marks
  2. If 2.2 is not working, check if matrix-a works. That is, check if, for IO operations, the VM can run alternatively or not. - 1/6 Mark
